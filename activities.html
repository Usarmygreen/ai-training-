🔍 Why This Is Wrong — 3 Critical Errors
  • ①
    LastLogonDate is NOT usable inside -Filter: The AD -Filter parameter uses a limited query language that cannot evaluate date comparisons using LastLogonDate. The script will throw an error or silently return 0 users. The correct approach: Get-ADUser -Filter * -Properties LastLogonDate | Where-Object { $_.LastLogonDate -lt (Get-Date).AddDays(-30) }
  • ②
    No -WhatIf flag — this WILL disable accounts immediately: In a real environment with hundreds of users, this script would disable service accounts, shared mailboxes, and admin accounts with zero warning. Always add -WhatIf and pipe to a review step first.
  • ③
    Send-MailMessage is officially deprecated (since PowerShell 7.1): It has no TLS 1.2/1.3 support and is flagged as a security risk. Modern replacement: Send-MgUserMail via the Microsoft Graph SDK, or route through an approved SMTP relay with explicit TLS settings.

"The Apple Silicon Trap" — Hardware Platform Mismatch

AI constantly applies Intel Mac fixes to Apple Silicon (M-chip) Macs. Catch it before the user wastes 20 minutes.

👥 Pairs ⏱️ 10 min 🟦 Rovo 🍎 Mac Focus ⭐ Common Real-World Scenario
PROMPT A: "A user has an Apple M2 MacBook Pro running macOS Sonoma. Bluetooth is not detecting any devices. Walk me through the complete fix step by step."

PROMPT B: "How do I reset the NVRAM on a 2023 MacBook Pro?"
macos-triage.sh — Apple Silicon Platform Audit
# Click one of the buttons above to simulate execution and audit the commands...
🔍 Why This Is Wrong — 3 Errors
  • ①
    SMC does NOT exist on Apple Silicon Macs (M1/M2/M3/M4): The System Management Controller was a physical chip on Intel Macs. It simply does not exist on any M-series Mac. Performing this key combo does absolutely nothing and wastes the user's time. Sending this to a user erodes their trust in you.
  • ②
    NVRAM reset via ⌘+⌥+P+R does NOT work the same on Apple Silicon: On M-chip Macs, NVRAM is reset automatically on every boot as part of the secure boot process. The key combination has no equivalent effect. Apple's official documentation confirms this.
  • ③
    The correct M2 Bluetooth fix was never mentioned: The actual fix is: Option+click the Bluetooth menu bar icon → "Reset the Bluetooth module" (macOS Ventura/Sonoma). Or: delete ~/Library/Preferences/com.apple.Bluetooth.plist and reboot. These are the documented Apple Silicon steps.

Tip: Click "Run Apple Silicon Bluetooth Fix" in the console emulator above to see the output of resetting plist configurations.

💬 Group Debrief Questions
  • If you sent the bad instructions to a user, how long would they spend on Step 3 before realizing it did nothing?
  • What prompt addition forces AI to specify which Mac generation it's giving instructions for?
  • What's your verification step before sending any Mac troubleshooting guide to a user?

Windows Troubleshooting · L1

"Legacy Command Trap" — Windows 7 Advice on Windows 11

AI training data skews toward older Windows versions. Catch outdated commands before they confuse users or fail silently.

👥 Groups of 3 ⏱️ 12 min 🟦 Rovo 🪟 Windows 11 Focus
PROMPT A: "A Windows 11 user's PC shows BSOD error 0xc000021a and won't boot. Give me the step-by-step fix."

PROMPT B: "How do I boot a Windows 11 machine into Safe Mode?"
windows-safemode.ps1 — BCD Boot Configuration Manager
# Click one of the buttons above to simulate execution and audit the commands...
🔍 Why This Is Wrong — 3 Errors
  • ①
    F8 at startup is DISABLED by default on Windows 10/11: Fast Startup + UEFI boot makes Windows boot too fast for F8 to register. This will fail for the vast majority of users. Correct methods: Shift+Restart → Troubleshoot → Advanced → Startup Settings, or bcdedit /set {default} safeboot minimal from an elevated CMD (if Windows still loads).
  • ②
    0xc000021a is a critical process death — sfc/scannow alone won't fix it: This BSOD means a critical user-mode process (winlogon.exe or csrss.exe) crashed fatally. SFC fixes corrupt system files, not crashed processes. The correct path: Boot from Windows 11 installation media → Startup Repair, or DISM /Image:C:\ /Cleanup-Image /RestoreHealth from WinPE.
  • ③
    chkdsk /r can take hours and may not be relevant: Running a full drive scan before ruling out other causes (driver corruption, recent update) wastes hours. Always check Event Viewer (via WinPE) for the specific process that crashed before prescribing disk repairs.

Tip: Click "Run Windows 11 Reboot to Recovery" in the console emulator above to see the correct modern command execution.

💬 Group Debrief Questions
  • A user follows the F8 instructions for 10 minutes and can't get Safe Mode. What's the support call like?
  • How do you prompt AI to always specify OS version compatibility for its instructions?
  • What's your internal verification resource for Windows troubleshooting? (Hint: docs.microsoft.com)

Networking · L1–L2

"Network Ghost" — Fabricated IPs & DNS Hallucinations

AI will confidently invent IP addresses and DNS server IPs. Deploy them and you break production. Find them first.

👥 Teams of 3–5 ⏱️ 20 min 🟦 Rovo 🌐 Networking 🏆 Competitive — find all 4 errors first
PROMPT A: "A Windows 11 user cannot resolve internal hostnames but can ping external sites by IP address. Walk me through the PowerShell diagnosis and fix for the DNS issue."

PROMPT B: "Give me the PowerShell command to set a static IP of 192.168.1.50, subnet 255.255.255.0, gateway 192.168.1.1, and primary DNS 8.8.8.8 on a Windows 11 PC."
dns-resolver.ps1 — Active Directory Network Triage
# Click one of the buttons above to simulate execution and audit the commands...
🔍 Why This Is Wrong — 3 Errors
  • ①
    157.58.0.0 is a FABRICATED IP — not a Microsoft DNS server: This address does not exist as a public Microsoft resolver. Configuring production machines with this address causes complete DNS failure. Microsoft's actual resolver (Azure DNS) is at 168.63.129.16 — and that's only reachable from within Azure VNets. Always verify DNS IPs against official Microsoft documentation or your own AD DNS servers.
  • ②
    Using Google 8.8.8.8 or external DNS for internal hostname resolution is wrong: Internal hostnames (like internalserver.corp) only resolve via your internal AD/DNS infrastructure. External DNS servers have no knowledge of your internal zones. This makes the problem worse, not better.
  • ③
    Missing the actual first steps of DNS triage: The real flow is: ipconfig /all → verify which DNS servers are assigned → Test-NetConnection -ComputerName [your-DNS-server] -Port 53 → then nslookup against your known-good internal resolver.
❌ Bad AI Output — Prompt B (Static IP)
New-NetIPAddress -InterfaceAlias "Local Area Connection" `
  -IPAddress 192.168.1.50 -PrefixLength 24 `
  -DefaultGateway 192.168.1.1

Set-DnsClientServerAddress -InterfaceAlias "Local Area Connection" `
  -ServerAddresses "8.8.8.8"
🔍 Why This Is Wrong — 1 Critical Error
  • ①
    "Local Area Connection" is a Windows 7/Vista-era interface name: Windows 10 and 11 name interfaces "Ethernet", "Ethernet 2", "Wi-Fi", etc. This command will fail with a silent error ("interface not found") or throw an exception. Before setting any IP config, always run Get-NetAdapter to get the actual interface alias name on this specific machine.

Tip: Click "Run local AD DNS resolution check" in the console emulator above to see the correct PowerShell lookup routing.

💬 Group Debrief Questions
  • If you pushed 157.58.0.0 as the DNS server to 50 workstations via GPO, what breaks and how fast?
  • What's your team's policy for verifying any IP address an AI gives you?
  • How do you prompt AI to always ask for your actual interface name before giving netsh/PowerShell commands?
  • What internal documentation source do you use to verify Windows networking commands?

Rovo / Jira · L1

"Ticket Triage" — Rovo for the Service Desk

Use Rovo inside Jira to summarize, respond to, and prioritize a realistic service desk ticket.

👥 Pairs ⏱️ 15 min 🟦 Rovo — Required 🎫 RTP Service Desk ⭐ L1 Core Skill

🎫 Mock Ticket — User Submitted

"Hi, my laptop is really slow lately and I can't get to my email. Also I think I clicked something weird in an email yesterday and now I have a bunch of popups. The wifi keeps disconnecting too. I need this fixed ASAP — I have a presentation at 2pm."

Step 1: "Summarize this ticket in 2 sentences for a team lead status update."

Step 2: "This ticket describes 3 separate issues. List them individually and suggest a priority (P1–P4) for each."

Step 3: "Draft a professional, empathetic first-response to this user. Acknowledge urgency, list what we'll investigate, and give one immediate self-service step. Keep it under 120 words."

Step 4 (bonus): "Based on the phishing indicator mentioned, what immediate containment steps should the service desk take before troubleshooting anything else?"
⚠️ Common Bad AI Behavior to Watch For
  • ①
    AI may bury the phishing indicator: The "I clicked something weird in an email" with popups is a potential security incident. AI often treats this as a performance issue. In a real ticket, this is a P1 isolation scenario — not a P3 performance ticket. Watch if Rovo catches this.
  • ②
    AI may give a generic response that ignores the 2pm deadline: Users told you there's a hard deadline. A good AI-assisted response acknowledges it. Check if the draft reflects urgency or uses boilerplate language.
  • ③
    AI may not separate the 3 issues properly: Slow laptop + possible malware + WiFi disconnects are three different ticket types. If Rovo combines them into one vague response, you've lost triage clarity.
💬 Group Debrief Questions
  • Did Rovo flag the phishing risk as a priority? Should it have?
  • How much did you have to edit the AI's draft response to make it sound like you?
  • Would you have triaged these as one ticket or three? What did Rovo suggest?
  • What's the ROI in time saved — and what's the risk if you don't review the output?

Prompt Crafting · All Levels

"The Prompt Olympics" — Iteration & PCTF Challenge

Teams compete to get the best output using the PCTF framework. 3 rounds, one scenario, one winner.

👥 Teams of 3–5 ⏱️ 20 min 🟦 Rovo or any AI 🏆 Competitive All Levels

📋 PCTF Reminder

P

Persona — who should AI act as?

C

Context — what background?

T

Task — what do you need?

F

Format — how to respond?

🎯 SCENARIO (same for all teams):
"A VIP executive's Windows 11 laptop cannot connect to the corporate VPN after a Windows Update ran last night. They have a board call in 45 minutes. Write a troubleshooting guide for the Tier 1 agent responding to this ticket."
  • 1

    Round 1 (4 min): Submit your instinct prompt with NO preparation. Record the output verbatim.

  • 2

    Round 2 (6 min): Apply the full PCTF framework. Rewrite and resubmit. How much better is it?

  • 3

    Round 3 (5 min): Add one specific constraint (e.g., "the user is non-technical," "give steps that take under 5 minutes," "flag any step that requires admin rights"). Resubmit.

  • 4

    Share-out (5 min): Each team reads their Round 3 output. Group votes: whose response would you actually send to a Tier 1 agent right now?

💬 Group Debrief Questions
  • What single change between Round 1 and Round 2 made the biggest difference in quality?
  • Which PCTF element (Persona, Context, Task, or Format) had the most impact for this scenario?
  • Did any team's Round 3 prompt produce a result you'd actually use without editing? Why or why not?
  • What's one prompting habit you're taking back to your desk tomorrow?
