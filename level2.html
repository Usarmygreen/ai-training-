Security & Governance

AI is powerful — but so is your responsibility. Understand the rules before you use any AI tool on work tasks.

🏢 AI Usage Policy — Two-Tier System

🔵 New L1 Employees

USE ATLASSIAN ROVO ONLY

Why Rovo? Enterprise-governed, data stays within the Atlassian ecosystem, auditable, and approved by IT. It is built into the tools you already use — Jira and Confluence.
❌ Do NOT use for work tasks:
🚫 ChatGPT (free or Plus account)
🚫 Claude.ai (free tier) or any consumer Claude plan
🚫 Google Bard / Gemini (non-Workspace version)
🚫 Any AI chatbot not approved by IT
Senior L1 / L2+ Employees

May Use Any AI Tool — With Full Responsibility

You have the experience to evaluate AI output critically. You may use Claude, ChatGPT, Gemini, or other tools — but the rules below are non-negotiable.
🚫 NEVER input into consumer AI:
🚫 PII or client personal data
🚫 Internal financials or budgets
🚫 Proprietary source code
🚫 Passwords or credentials
🚫 Client contracts or SLAs
🚫 Trade secrets or IP
📋 Document AI usage when it substantially contributes to a deliverable. Vet ALL outputs before use — AI can be confidently wrong.
Fluency Spillover: Developing your AI fluency outside the office is highly encouraged. Using AI chatbots at home for personal projects, hobbies, trip planning, or drafting content builds the core "prompt muscle memory" that directly makes you more efficient and confident when using Atlassian Rovo at work.

Best Practices — Click to Expand

AI output is a first draft, not a final product. Always read through the response before using it anywhere. Check for factual errors, hallucinated links, incorrect syntax, and tone. A 30-second review can prevent a bad ticket, a broken script, or misinformation being forwarded to a client.

If AI substantially wrote, drafted, or structured something you are delivering as work product — a Confluence doc, an email, a runbook — note it. This builds trust, helps others understand what to verify, and keeps your team accountable to quality standards.

AI models can hallucinate URLs, cmdlet names, API parameters, and dates. Before you paste a PowerShell script, check the syntax against official Microsoft docs. Before you reference a KB article number, confirm it exists. No exceptions.

Structure every prompt with Persona, Context, Task, and Format. This gives the AI enough information to produce a targeted, relevant response and reduces the chance it makes something up to fill gaps. Better prompts = better, safer outputs.

If Rovo or any AI gives you advice that feels off, risky, or plainly wrong — flag it. Tell your team lead. Document what happened. AI tools are still maturing, and collective feedback makes the whole team smarter. If you encounter a security-relevant AI response, escalate immediately.

Advanced Workflows — Case Studies

Real-world scenarios for the RTP Service Desk. Adapt these patterns to your daily work.

Case Study 01 · Security-First
🔒

AI Code Refactor — Safely

Need AI to review or improve a PowerShell script? You can do this without exposing sensitive logic, credentials, or internal environment specifics. The key is sanitizing before submitting.

  • 1. Remove all server names, IPs, and internal paths
  • 2. Replace credentials with values
  • 3. Ask for a code review, not execution
  • 4. Validate output against official PowerShell docs before running
Prompt Template
"You are a senior PowerShell engineer. Review this script for errors, performance issues, and best practices. [PASTE SANITIZED SCRIPT]. Do not run it — explain what each section does and flag anything risky."
Case Study 02 · Rovo Workflow
📋

Meeting Summary Automation

After a sprint review or incident debrief, use Rovo inside Confluence to instantly convert raw meeting notes into clean, actionable summaries — no manual rewriting required.

  • 1. Open the Confluence meeting notes page
  • 2. Invoke Rovo from the Confluence toolbar
  • 3. Paste or point Rovo to the notes content
  • 4. Review the output and add it as a child page
Example Rovo Prompt
"Summarize these meeting notes into: 1) Key decisions made, 2) Action items with owner and due date, 3) Open questions still pending. Format as a table where possible."
Case Study 03 · Documentation
📝

Documentation from Bullets

Got rough bullet points from an incident or a new runbook request? AI can draft the full structured document from your shorthand notes — dramatically cutting documentation time.

  • 1. Write down your bullet-point facts and steps
  • 2. Remove any sensitive data before using consumer AI
  • 3. Use Rovo for internal docs (safe by default)
  • 4. Review output carefully and add real-world specifics
Example Prompt
"You are an IT documentation specialist. Convert these bullet points into a professional incident report with sections: Summary, Timeline, Root Cause, Resolution, and Prevention. Bullets: [paste your notes]"
Remember: AI is a multiplier, not a replacement. The best outcomes come when you bring your domain knowledge and judgment to guide the AI — then verify its output before it goes anywhere near a client or production system.
