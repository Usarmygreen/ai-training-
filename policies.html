Data Security Classification Guidelines

All company information is categorized into three compliance classifications when communicating with LLM systems.

CLASS A: PUBLIC & GENERAL (SAFE)

Approved for processing across Rovo, Claude, or Gemini. Focuses on logic and syntax helper commands.

  • Generic programming syntax (e.g. loops, arrays)
  • Regular expression formatting questions
  • Sanitized event logs (with server IPs removed)
  • Public API endpoint parameters & parameters help
  • Standard Windows/macOS shell commands
  • Basic documentation formatting (Markdown tags)
CLASS B: INTERNAL & CONFIGURATION (RESTRICTED)

Must be thoroughly sanitized before inputting. Stripping all specific context indicators is required.

  • Internal Active Directory domain structures
  • Private subnet ranges (e.g. 10.0.0.0/8)
  • Specific server hostnames (e.g. DC-01.corp)
  • Sanitized custom PowerShell automation scripts
  • Software installation path directories
  • Non-public internal ticketing log headers
CLASS C: CONFIDENTIAL & PROPRIETARY (FORBIDDEN)

DO NOT UNDER ANY CIRCUMSTANCES INPUT INTO PUBLIC CHATBOTS. Serious security incident risk.

  • Customer Personally Identifiable Information (PII)
  • Passwords, access tokens, API credentials
  • Internal financial records and team budgets
  • Client contracts, SLAs, or proprietary codebases
  • Active security exploit traces or threat logs
  • Corporate email communications text

Code Vetting Checklist

Always verify AI-generated scripts or commands against this checklist before executing them in production or testing environments.

  • 1. Parameter check: Verify every parameter in the cmdlets or shell script. Look up unfamiliar parameters in the PowerShell Docs.
  • 2. Dry Run (-WhatIf flag): If the cmdlet makes changes (e.g. disabling, deleting, writing configurations), ensure a -WhatIf or -Confirm argument is piped first.
  • 3. Deprecation audit: AI training data may skew older. Confirm cmdlets (like Send-MailMessage) are not deprecated and match modern secure standards.
  • 4. Interface alias mismatch: Check hardware-specific names (e.g. InterfaceAlias Wi-Fi vs Ethernet). Ensure instructions are applicable to the user's specific system (Windows 11 vs Apple Silicon).
  • 5. IP Verification: Ensure all DNS IPs or gateway addresses are cross-referenced with internal documentation, rather than using model-generated ghost IPs.
Compliance Incident? If you accidentally input Forbidden (Class C) data into a public consumer AI chatbot, do not try to delete the chat (which doesn't purge logs from corporate databases). Note down the timestamp, the prompt input, and report the incident immediately to your security lead.
